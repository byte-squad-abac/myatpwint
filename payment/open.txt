=== TRANSACTION TYPES ===

MPU Gateway:
1. Payment Request/Response - Sale transactions
2. Inquiry Request/Response - Transaction status check
3. Void Request/Response - Cancel approved transactions

KBZPay Gateway:
1. kbz.payment.precreate - Create payment order (PWAAPP)
2. kbz.payment.queryorder - Query order status
3. kbz.payment.queryrefund - Query refund status
4. Callback Interface - Payment result notifications

=== INTEGRATION METHODS ===

PWAAPP Flow:
1. Merchant calls precreate API with trade_type=PWAAPP
2. Get prepay_id from response
3. Generate signature with order info
4. Redirect to KBZPay PWA page with signed parameters
5. KBZPay app launches for payment
6. Receive callback notification on success
7. Query order status if needed

=== IMPORTANT NOTES ===

- All amounts must be padded with '0' and no decimal point (e.g., 1.00 = 000000000100)
- Invoice numbers must be unique within current day
- Hash signature required for all requests using merchant secret key
- Supports both UI and Non-UI integration methods
